Why is it necessary to enter a link to a webpage, and not to an image ?

Because it is necessary to find the artist's Twitter account(s). Indeed :

  • It is not possible to index all images posted on Twitter. Plus, it would build a 99% useless database.
  • And it's impossible to distinguish artists' Tweets to Tweets from people who repost their illustrations.

Thus, we must analyze the source of the illustration to find the Twitter account(s) of the artist of the illustration.

This analysis is therefore different for each source website. So different algorithms are needed for each website. This is why there is a restricted list of supported websites.

I have an illustration, but no link (No source). What can I do ?

As explained above, Artist on Twitter Finder cannot perform a search directly with an image file.

Here is a list of image search engines that might help you find where an artist has published an illutration :

  • SauceNAO, specialized in artists' illustrations. It is thus very precise in its results.
  • TinEye, generalist, but tries to highlight the source websites of the images.
  • Yandex, generalist, also works with modified or cropped images.
  • Google Images, same comments as for Yandex.

Why can illustrations take a long time to be processed ?

There are two main limitations :

  • The rate limits on Twitter's APIs. And they are very drastic.
  • Image analysis time.

Thus, if it is necessary to index or update the indexing of a Twitter account, the processing time of the illustrations will be very long. And this time increases with the number of illustrations being processed in the server.

How are artists' Twitter accounts searched ?

The search for Twitter accounts consists in analyzing web pages looking for links (URL). The AOTF server looks for links on the illustration's webpage, as well as on the artist's profile. If a link leads to a Twitter account, it will be considered as belonging to the artist. Moreover, if a link leads to a profile on another supported site, it will also be considered as belonging to the artist, and the search for Twitter accounts will be recursive.

If multiple Twitter accounts are found, they will all be considered as the artist's.

Details on Twitter accounts search for each supported website :

  • DeviantArt : In the description of the illustration and in the artist's biography on his/her profile.
  • Pixiv : In the links indicated by the artist on his profile.
  • Boorus (Danbooru, Derpibooru, Furbooru) : In the informations about the artist, as well as the source of the illustration, if it comes from a supported website. Warning : If the source is a Tweet, the associated Twitter account is not considered as the artist's Twitter account (Except for sources on Danbooru).

Notes :

  • The search for Twitter accounts is done independently for each illustration. No association is stored.
  • If the Twitter accounts found are non-existent or deleted, suspended, deactivated, or private, they will be considered as invalids, because no indexing or search can be made. In addition, AOTF has a blacklist of accounts that are listed for containing a lot of Tweets with media, without being artist accounts. If an account found is on this list, it will also be considered as invalid.
  • If a link leads to a profile on one of the following websites, this profile will also be scanned for Twitter accounts : Linktree, Patreon and Carrd.

I am an artist, I have a Twitter account, but it is not found. Why ?

Make sure you've put a link to your Twitter account where the server is searching (See above).

What data is stored ?

The following data is stored in the database :

  • Twitter accounts : ID of the Twitter account, and index cursors for this account.
  • Tweets : ID of the Tweet, ID of the associated Twitter account, IDs and fingerprints of associated images.

Notes :

  • It is impossible to "rebuild" images from fingerprints.
  • Tweets without an image are not indexed.
